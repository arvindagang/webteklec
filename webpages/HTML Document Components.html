HTML Document Components

Document Type Declaratios (DOCTYPE)

HTML 4.01 <DOCTYPE>

 ➧HTML 4.01 Strict Document Type Definition (DTD)
  Strict Document Type Definition excludes the elements and attributes that W3C expects to remove permanently as support for style sheets.
 ➧HTML 4.01 Transitional
  Transitional Document Type Definition that include the elements and attributes that W3C expects to remove permanently as support for style sheets.
 ➧HTML 4.01 Frameset
  Frameset Document Type Definition that is used for documents that has frames.

XHTML 1.0 <DOCTYPE>

 ➧XHTML 1.0 Strict
  Extensible HTML version 1.0 Strict DTD gets rid of presentational labels and qualities, and is in fact extremely strict.
 ➧XHTML 1.0 Transitional
  Extensible HTML version 1.0 Transitional DTD in which it is the same as HTML 4.01 Strict but for some exception that differs between SGML and XML.
 ➧XHTML 1.0 Frameset
  Extensible HTML version 1.0 Frameset DTD utilized when Frames are to partition the browser into at least two frames. The Frameset DTD incorporates everything in the transitional DTD in addition to frames.

*Elements (HTML 4.0, XHTML 1.0, HTML 5)

 ➧Tags
 Are to classify the contents in a web page. It allow unstructured classification, means that the reader can track the content better.

 ➧Contents
 Content Model
  Used to characterize the sort of substance that can be found inside a component in HTML. Generally, this is characterized in the DTD for a SGML-inferred language

 ➧Void Elements
 Element that must not have content. They are:

<br> <hr> <img> <input> <link> <meta> <area> <base> <col> <command> <embed> <keygen> <param> <source> <track> <wbr>.
