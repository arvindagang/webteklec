Resource Addressing

 *Scheme
  Identifies the protocol to be used in accessing the resource on the Internet. It may be HTTP or HTTPS

 *Host name
  - an identifier that holds the resource
  - also known as domain name
  - it is provided by the server. It is mapped to a specific IP address, but host names and IP addresses don't have a one-to-one relationship

Continuation

 *Path
  A path identifies the specific resource in the host that the client wants to access.
   eg. /var/www/index.html

 *Query
  - whenever a query string is used, it follows the path and provides information that the resource can use for other purpose like parameters when searching, or data that will be processed.
  - is usually a string of name and value pairs.
