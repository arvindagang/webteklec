Continuation

 Safe Methods - are methods that does not modify server resources.
 Indempotent Methods - are methods use to manipulate a sever resources.

  Methods or verbs:
  GET - request for specified resource. SAFE-INDEMPOTENT
  HEAD - request for the head content. SAFE-INDEMPOTENT
  POST - submit something in a resource.
  PUT - replace something you put. INDEMPOTENT
  DELETE - delete a specified resource. INDEMPOTENT
  CONNECT - establish a connection.
  OPTION - option for target resource. SAFE-INDEMPOTENT
  TRACE - create a message loopback for the target resource.
  PATCH - apply modification to a resource.

Content-Negotioation and other necessary thing.

 Going deeper, much necessary information are the content-negotiation response by the header, it is use with Accept, Accept-Langauge, Content, and Content-Langauge, these are headers or metadate which manipulates the documentary of the site.

 Accept - Syntax "Accept: $param" following parameter: text/html, image, */* , text/plain This will send back document format according to the inputed parameter.

 Accept-Langauge - Syntax "Accept-Langauge: en-US" following parameter: en-US, dan, en-UK, fil, fr and more. The default langauge is always us, however, if the langaug is not found another langauge is subtitute. langauge has its own queue for langauge subtitute.

 Content-Langauge - Syntax "Content-Languege: $param" following parameter: same as the above. Return the content langauge. The content totally depends on the specifed langauge.
